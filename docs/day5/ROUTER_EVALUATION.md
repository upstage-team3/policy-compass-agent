## 공통 합격 기준

- `routing_source=llm`인 정상 Router 결정을 키워드 규칙이 덮어쓰지 않는다.
- `SEARCH`는 선택된 외부 데이터 Tool 하나만 실행한다.
- `RESPOND`는 외부 데이터 Tool을 실행하지 않는다.
- 정책명, 기관, 금액, 날짜, 신청 조건과 링크는 API 후보 범위를 벗어나지 않는다.
- 결과가 없거나 API가 실패하면 데모 데이터를 만들지 않고 한계를 안내한다.
- SSE는 `status → status → token → done` 또는 오류 시 `status → error` 흐름을 유지한다.
- API 키, 인증 query string, 주민등록번호·카드번호 형태가 로그와 캡처에 노출되지 않는다.

## 새로고침·대화 기록 회귀

1. 주거정책 질문을 입력해 지역·나이 확인 질문까지 진행한다.
2. 브라우저를 새로고침하고 기존 채팅과 확인 질문이 그대로 보이는지 확인한다.
3. `서울에 사는 만 25세야`라고 답해 원래 주거정책 검색을 재개하는지 확인한다.
4. 새 채팅을 추가한 뒤 다시 새로고침해 채팅 목록과 활성 채팅이 유지되는지 확인한다.
5. 개별 채팅과 전체 기록 삭제가 새로고침 뒤에도 유지되는지 확인한다.
6. 지역·나이를 확인한 채팅에서 새 채팅을 만든 뒤 13번 질문을 입력하고 조건 재질문 없이 추천되는지 확인한다.
7. 전체 기록 삭제 후 새 채팅에서는 이전 지역·나이가 더 이상 자동 적용되지 않는지 확인한다.

브라우저에는 최근 채팅 20개·채팅별 메시지 50개까지만 저장되며, 주민등록번호·카드번호 형태는 저장 전에 `[민감정보 삭제]`로 바뀌어야 한다.
