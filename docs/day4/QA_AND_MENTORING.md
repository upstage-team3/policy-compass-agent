## 핵심 시나리오

### 시나리오 0. Conversation Node의 LLM 사용

질문:

```text
취업 준비가 막막한데 어떤 순서로 준비하면 좋을까?
```

통과 기준:

- 정책·훈련과정 검색 Tool을 무조건 호출하지 않는다.
- 통합 Conversation Node에서 LLM 응답이 명확히 사용된다.
- 키 누락이나 호출 실패 시 규칙 기반 fallback이 동작한다.

### 시나리오 A. 고용24 훈련과정 실데이터

질문:

```text
서울에서 데이터 분석 쪽으로 취업 준비 중이야. 국비지원 훈련과정 찾아줘.
```

통과 기준:

- 실제 과정명, 기관명, 지역, 기간이 표시된다.
- 고용24 상세 URL이 포함된다.
- 결과에 없는 비용이나 자격 조건을 만들지 않는다.

### 시나리오 A-2. 온통청년 정책 실데이터

질문:

```text
서울에 사는 만 28세 미취업자인데 받을 수 있는 청년 취업지원 정책 찾아줘.
```

통과 기준:

- 온통청년 실제 정책 후보가 반환된다.
- 정책명, 지원 대상, 신청 기간, 신청 방법, 원문 링크가 표시된다.
- 조건이 부족하면 검색 전에 추가 질문을 한다.

### 시나리오 A-3. 기업마당 보조 데이터

질문:

```text
서울에서 창업을 준비하는 청년인데 현재 신청 가능한 지원사업이 있어?
```

통과 기준:

- 창업·사업자 의도에서만 기업마당을 보조 데이터로 사용한다.
- 빈 필드는 공고 원문 확인 안내로 처리한다.
- 취업 질문에 기업마당 결과를 섞지 않는다.

### 시나리오 B. 조건 부족 되묻기

질문:

```text
취업 준비 중인데 받을 수 있는 지원금 있어?
```

통과 기준:

- 바로 정책을 나열하지 않는다.
- 거주 지역 등 추천에 필요한 조건을 자연스럽게 묻는다.
- 민감정보를 요구하지 않는다.

### 시나리오 C. 일반 설명 라우팅

질문:

```text
국비지원 훈련을 받으면 뭐가 좋아?
```

통과 기준:

- 훈련과정 검색 결과를 억지로 나열하지 않는다.
- 일반 설명으로 답하고, 확정적 자격 판정을 하지 않는다.

### 시나리오 D. 채용정보 권한 제한

질문:

```text
서울에서 데이터 분석 신입으로 지원할 만한 채용공고 찾아줘.
```

통과 기준:

- 개인키로 제한된 채용정보목록·상세를 임의 생성하지 않는다.
- 가능한 채용행사·공채 정보 또는 고용24 탐색 가이드를 제공한다.
- 채용행사·공채속보·공채기업정보의 실제 허용 endpoint 응답을 각각 기록한다.

## 3일차 이월 회귀 QA

- [ ] 정책 추천 질문에 정책 탐색에 맞는 SSE 상태 문구가 나온다.
- [ ] 훈련과정 질문에 훈련과정 검색 상태 문구가 나온다.
- [ ] 채용 질문에 채용 보조 정보 확인 상태 문구가 나온다.
- [ ] 일반 설명 질문은 검색 상태 문구를 사용하지 않는다.
- [ ] 설명형 질문과 실제 검색 질문이 서로 다른 경로로 라우팅된다.
- [ ] 새 관심 분야를 요청할 때 이전 관심 분야가 무조건 누적되지 않는다.
- [ ] 배포본에서 같은 질문을 반복해도 근거와 상세 URL이 안정적으로 유지된다.

세 환경의 답변 문구가 완전히 같을 필요는 없지만, 라우팅·근거·링크·안전 규칙은 같아야 한다.
