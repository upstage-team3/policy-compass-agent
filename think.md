# Day3 Think Note

작성일: 2026-07-10

## 오늘의 핵심 판단

Day3의 중심은 새 기능 확장보다 기존 Agent 흐름을 발표 가능한 수준으로 안정화하는 것이다.
따라서 LLM이 정책 사실을 새로 만들지 못하게 막고, 조건이 부족한 질문에서는 추천보다 추가 질문을 먼저 하며, SSE/UI가 사용자를 기다리게 하지 않는 방향으로 개선했다.

## 트러블슈팅

### LLM 사실 생성 위험

LLM은 후보 정책을 자연어로 설명하는 역할로만 제한해야 한다. 프롬프트에 후보 데이터 밖의 정책명, 금액, 날짜, 자격 조건, 신청 방법, 링크를 만들지 말라는 규칙을 명시했다.
템플릿 fallback도 같은 구조를 따르도록 보강해 LLM이 꺼져 있어도 답변 품질이 흔들리지 않게 했다.

### 조건 부족 상태의 오추천

`취업 준비 중인데 받을 수 있는 지원금 있어?` 같은 질문은 거주 지역이 없어 추천 정확도가 낮다.
이 경우 바로 정책을 나열하지 않고 거주 지역 등 부족 조건을 먼저 묻는 동작을 회귀 테스트로 고정했다.

### 기업마당 API 응답 필드 누락

기업마당 응답은 실제 공고별로 대상, 지원 내용, 신청 방법, 원문 링크가 비어 있을 수 있다.
정규화 단계에서 빈 필드는 "공고 원문 확인 필요" 문구로 대체하고, 원문 링크가 없으면 기업마당 공식 사이트로 연결하도록 했다.

### SSE 빈 말풍선

기존 UI는 Agent 실행이 끝나기 전까지 빈 말풍선이 보일 수 있었다.
스트리밍 시작 시 `status` 이벤트를 먼저 보내고, UI는 첫 토큰을 받으면 상태 문구를 실제 답변으로 교체하도록 개선했다.

### Docker 검증

`docker compose config`는 유효했지만 `env_file: .env` 때문에 설정 출력에 실제 키가 펼쳐질 수 있다. 이 출력은 공유하지 않는 것이 안전하다.
`docker compose build`는 Docker Desktop 데몬이 실행 중이지 않아 실패했다. Docker Desktop 실행 후 빌드와 기동 검증을 다시 해야 한다.
